How To Create a Plugin

There are many ways a developer can create and load a PDI plugin but for now we will just highlight the basics of how it can be done.

To create a plugin you only need to extend the RichPlugin class as shown below:

<?php
class MyPlugin extends RichPlugin {

    public function doSomething() {
        // some code here
    }
}
?>

Next, you will need to let the framework know whether or not you would like the plugin's render function to be called whenever the page is being rendered. To do this you only need to set declare and set the class variable $implementPrerender.

<?php
class MyPlugin extends RichPlugin {

    protected $implementPrerender = true;

    public function doSomething() {
        // some code here
    }

    public function render() {
        // render plugin code
    }
}
?>

When you're finished you can save your plugin to the raxan plugins folder. For example you could save the file (dosomething.php) to a folder called myplugins (raxan/plugins/myplugins).

Loading a plugin

To load a plugin you simple make a call the the RichAPI::loadPlugin() method:

<?php
    RichAPI::loadPlugin('myplugins/dosomething');
?>

The above will load the plugin relative to the plugins.path config setting, which defaults to the raxan/plugins folder. Notice that we did not have to specify the .php extension for the plugin file.

To load a plugin that's located inside an alternate folder (not relative to plugins.path):

<?php
    RichAPI::loadPlugin('/path/to/plugin/dosomething.php', true);
?>

The second parameter will cause the framework to load the plugin based on specified the folder path and filename. Please notice that we had to include the .php extension when loading plugins that are external to the assigned 'plugin.path'.
